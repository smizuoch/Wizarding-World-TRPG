## 追加のサイト挙動修正

| 対象 | 問題 | 対応 |
| --- | --- | --- |
| 初期表示 | ブラウザが `favicon.ico` を要求し、404がConsoleエラーとして出ていた。 | インラインfaviconを追加。 |
| ココフォリア出力 | `立ち絵URL` を入力しても `iconUrl` が空のままだった。 | `profile.imageUrl` を `iconUrl` へ反映。 |
| 保存データ読み込み | 保存済みの `今日のひとこと` が読み込み時に当日の文言で上書きされていた。 | 保存値がある場合は保存値を復元。 |
| 保存データ読み込み | 読み込みデータに `祝福`、`呪い`、`幸運` がない場合、直前に表示していた値が残る可能性があった。 | 祝福25、呪い0、幸運空欄へ明示的に初期化。 |
| ローカル保存 | `localStorage` の保存リストが配列以外に壊れていると、初期化時に落ちる可能性があった。 | 配列でない保存値は空リストとして扱う。 |
| 回覧モード | 回覧モード中でも一部の編集系ボタンが有効で、disabled入力へJS経由で書き込めた。 | 保存、新規作成、能力値セット入力、チャットパレット自動作成などの変更系操作を無効化。 |
| ココフォリアコピー | 回覧モード中のコピー操作で、副次ステータスやチャットパレットが自動生成されて内容が変わる可能性があった。 | 回覧モード中は現在の出力内容をコピーするだけに変更。 |
| カスタム技能削除 | カスタム技能を削除しても、プレビューとココフォリア出力が次の入力まで更新されなかった。 | 削除直後にプレビュー更新を予約。 |
| JSON読み込み | 同じJSONファイルを続けて選択した場合、ブラウザによって再読み込みできない。 | 読み込み後にファイル入力をリセット。 |
